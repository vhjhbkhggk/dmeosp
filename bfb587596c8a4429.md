# MQS Archive Bridge

MQS Archive Bridge 是一个面向技术文档、代码片段与开发日志的轻量化外链聚合与导航系统。项目定位于帮助开发者、技术写作者与团队负责人从分散的代码仓库、临时笔记和碎片化记录中快速检索、分类与引用具体的提交级文档或说明文件，避免在大量历史记录中反复定位。

项目不提供内容存储，而是提供一套基于 URL 模式的索引与浏览框架，支持将深层 GitHub 文件链接组织为可读、可维护的资源目录。目标用户包括需要管理大量技术参考资料的个人开发者、维护多仓库文档索引的团队，以及希望将临时分析记录结构化的研究人员。

## 功能概览

**结构化链接索引**：将大量原始文件链接按项目批次与编号进行归类和展示，支持按文件名模式快速筛选。

**只读仓库视图**：所有收录链接指向特定仓库的只读文件版本，确保引用内容不可变，避免分支更新导致参考失效。

**批量导入与更新**：提供脚本化工具，支持从文本文件或 CSV 批量导入链接列表，自动解析仓库路径与文件名。

**标签与注释系统**：允许用户为每个链接添加自定义标签和备注，记录该链接对应的技术主题、关联问题或使用状态。

**搜索与过滤接口**：内置基于文件名、标签和注释的简单搜索功能，支持按批次、日期范围和文件扩展名过滤。

**导出与分享功能**：支持将当前索引列表导出为 Markdown 表格、JSON 或纯文本格式，便于嵌入其他文档或分享给团队成员。

## 应用场景

**技术文档归档**：团队在开发过程中产生大量设计说明、评审记录和问题分析文档，分散在不同仓库分支。使用 MQS Archive Bridge 将这些文档的固定版本链接汇总到一个索引页面，方便新成员快速查阅历史决策依据。

**代码审查参考**：代码审查过程中，审查者需要引用先前提交中的示例代码或修复记录。通过本项目的索引结构，审查者可以快速定位到具体文件链接，并在审查评论中直接引用，减少查找时间。

**学习笔记管理**：个人开发者学习新技术时，常将关键代码片段或配置示例保存为单独的 Markdown 文件。本项目提供的索引框架可以将这些文件的链接组织为按主题分类的目录，形成可检索的个人知识库。

**离线文档预备**：在无法访问完整仓库的环境下，通过预先导出的链接列表，可以批量下载对应的文件内容作为离线参考资料，保证工作连续性。

## 快速开始

以下步骤将项目克隆至本地并启动基础索引服务。

```bash
git clone https://github.com/your-org/mqs-archive-bridge.git
cd mqs-archive-bridge
pip install -r requirements.txt
python scripts/build_index.py --input data/links.txt --output index.json
```

如需启动本地 Web 浏览界面，可运行：

```bash
python app.py --port 8080
```

访问本地端口即可查看索引后的链接列表与搜索界面。

## 贡献指南

1. 在 GitHub 上复刻本项目至个人账户，并克隆到本地开发环境。创建新分支时请使用 feature/ 或 fix/ 前缀，例如 feature/add-csv-export。

2. 运行测试套件确保当前代码通过全部测试。新增功能或修复问题后，请补充对应的单元测试用例，保证测试覆盖率不低于 80%。

3. 提交代码前使用 black 和 mypy 进行格式化和类型检查。提交信息请遵循约定式提交规范，使用 feat:、fix:、docs:、refactor: 等前缀。

4. 对于新增的导入格式、导出格式或搜索过滤条件，请同步更新 docs/ 下的对应文档，并在 user-guide.md 中添加使用示例。

5. 发起 Pull Request 到主仓库的 main 分支，描述中请说明变更内容、影响范围以及相关的 issue 编号。PR 需要通过 CI 检查后方可合并。

## 常见问题

问：索引中的链接指向的文件被删除或重命名后如何处理？

答：本项目仅维护链接索引，不托管文件内容。若发现链接失效，可在 data/links.txt 中注释或移除对应行，然后重新运行 build_index.py 更新索引。建议定期检查链接可用性，可使用 scripts/check_links.py 工具进行批量验证。

问：如何为链接添加自定义标签或注释？

答：在 data/tags.csv 文件中按格式填写记录，每行包含链接文件名、标签列表（逗号分隔）和注释文本。重新构建索引后，标签和注释将出现在 index.json 中。Web 界面也支持通过界面添加标签，修改将保存至 tags.csv。

问：搜索功能支持哪些字段和语法？

答：搜索默认匹配文件名、标签和注释字段。支持多个关键词以空格分隔，表示逻辑与关系。可用 tag: 前缀限定标签范围，用 file: 前缀限定文件名范围。例如 "tag:python file:config" 表示搜索带有 python 标签且文件名包含 config 的链接。
