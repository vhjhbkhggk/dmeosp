# LinkForge Index

LinkForge Index 是一个面向开发者、技术研究人员与开源贡献者的结构化外链与资源索引系统。项目定位于对高价值技术内容进行人工筛选、分类归档与语义化组织，帮助用户从海量信息中快速定位所需文档、教程、工具与参考实现。

项目核心目标用户包括：正在学习特定技术栈的初学者、需要查阅底层实现细节的进阶开发者、以及希望追踪生态动态的技术决策者。LinkForge Index 不提供托管存储，而是通过严格的链接审核机制与层级化目录树，构建可维护、可扩展的技术资源图谱。

## 功能概览

层级化目录结构 每个资源条目按主题域、子域与粒度三层路径归档，支持直观的浏览与回溯。

哈希内容寻址 所有资源条目均以 SHA-1 哈希值作为文件名存储，确保内容唯一性并防止重复提交。

状态标记系统 每条记录附带审核状态、最后验证时间与可用性标签，便于识别失效链接。

全文检索支持 内置基于标题与标签的模糊搜索能力，可通过命令行或 Web 接口调用。

批量导入与校验 支持通过 CSV 或 JSON 格式批量导入链接列表，自动执行可达性与内容类型检测。

变更历史追踪 所有增删改操作记录在变更日志中，支持回滚与审计。

社区反馈集成 允许用户通过 Issue 或 PR 提交补充信息、错误修正与权重建议。

## 应用场景

技术选型调研 架构师在评估多个候选方案时，可通过 LinkForge Index 快速获取各项目的官方文档、性能测试报告与社区讨论聚合，大幅缩短信息收集周期。

文档撰写参考 技术作者在撰写教程或博客时，需要引用权威资料。本索引提供经过初步筛选的参考链接，降低引用错误或链接失效的风险。

离线知识库构建 企业内网用户可通过批量导出功能，将索引中的外链元数据与缓存快照整合至内部知识管理系统，实现隔离环境下的文档查阅。

开源项目依赖梳理 维护者在审查依赖库的安全性与活跃度时，可利用本索引的标签体系横向对比多个备选库的维护状态与版本发布频率。

## 快速开始

以下指令适用于 Linux / macOS / Windows WSL 环境。

```bash
git clone https://github.com/linkforge/index.git
cd linkforge-index
pip install -r requirements.txt
python cli.py init --storage ./data
python cli.py serve --port 8080
```

首次运行将自动创建默认目录结构与示例索引条目。Web 界面默认监听 127.0.0.1:8080，可通过浏览器访问。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 或更高 | 核心运行时，用于 CLI 与调度服务 |
| pip | 22.0 或更高 | 管理 Python 包依赖 |
| Git | 2.25 或更高 | 用于克隆仓库与提交变更记录 |
| SQLite | 3.35 或更高 | 存储条目元数据与标签关系 |
| curl | 7.68 或更高 | 用于远程链接可达性检测 |
| redis | 6.0 或更高（可选）| 启用缓存后提升查询性能 |

## 贡献指南

1. 查阅问题追踪器 在提交任何代码或资源之前，请先浏览 GitHub Issues 中标记为 "help wanted" 或 "resource-request" 的工单，避免重复工作。

2. 创建功能分支 从 main 分支切出新分支，命名遵循 feat/功能描述 或 fix/问题简述 格式，例如 feat/add-kubernetes-resources。

3. 更新条目列表 若为新增链接，需在 data/index.db 中插入记录，并运行 cli.py validate 确保所有字段符合 schema 定义。若为修改已有条目，需在变更日志中注明原因。

4. 执行测试套件 在提交 Pull Request 前，请于本地运行 pytest tests/，确保所有单元测试与集成测试通过，且代码覆盖率不低于 85%。

5. 提交拉取请求 推送到远程仓库后，创建 Pull Request 并填写模板中的检查清单。至少需要一名维护者审核通过后合并。

## 常见问题

Q: 索引中的链接失效了怎么办？

A: 系统每天凌晨执行一次自动可达性检查。若检测到链接返回 4xx 或 5xx 状态码，会将条目标记为 "unreachable" 并发送通知给最后修改人。用户也可手动运行 cli.py check --entry <hash> 触发即时验证。若链接永久迁移，请提交 PR 更新 URL 字段。

Q: 如何请求添加新的资源分类？

A: 请在 GitHub Discussions 中发起 "分类提案" 类型的话题，说明新分类的名称、描述、预期包含的子域以及与其他现有分类的边界。经社区讨论并由核心维护者投票通过后，会在下一个版本中新增该分类。

Q: 是否支持私有化部署？

A: 支持。整个系统基于 SQLite 与 Flask 构建，无外部商业依赖。用户可下载源码后修改 config.yaml 中的存储路径与监听地址，即可在内网环境中完全独立运行。redis 缓存为可选组件，不启用时不影响基本功能。
