# MQSAQL External Reference Index

MQSAQL is a curated external reference index and technical resource aggregation system designed for developers, researchers, and technical writers who need to maintain organized references to distributed documentation fragments. The project provides a structured catalog of external markdown resources hosted across version control systems, enabling teams to track, reference, and integrate external technical content into their own documentation pipelines.

The system addresses the common challenge of managing large numbers of external reference files that are not under direct version control of the consuming project. By maintaining a centralized index with deterministic file naming conventions and metadata tracking, MQSAQL allows users to establish reliable foreign key relationships between their internal documentation and externally hosted technical content. The project is particularly suited for organizations that produce technical documentation, API references, or knowledge bases that depend on third-party or distributed content sources.

## 功能概览

**External Reference Indexing** - Provides a comprehensive catalog of external markdown resources with cryptographic hash-based filenames to ensure content integrity and avoid naming collisions.

**Deterministic Content Addressing** - Each external reference is addressed via its SHA-derived filename, enabling reproducible builds and content validation across different environments.

**Metadata Extraction Pipeline** - Extracts and stores frontmatter, headings, and structural information from each referenced markdown file to support search and discovery operations.

**Batch Processing Support** - Handles large-scale imports of external references in configurable batch sizes with progress tracking and error recovery mechanisms.

**Command-line Interface** - Offers a CLI tool for adding, updating, removing, and querying external references without requiring manual editing of the index.

**Conflict Resolution Strategies** - Implements last-write-wins and manual review policies for handling content updates and version discrepancies.

**Export Formats** - Supports exporting the reference index in JSON, YAML, and CSV formats for integration with external documentation systems and data analysis tools.

**Integrity Verification** - Performs checksum validation on referenced content to detect modifications and alert users to potential content drift.

## 应用场景

**Documentation Teams Managing Distributed Content** - Technical writing teams can use MQSAQL to maintain a central registry of external reference files that are incorporated into larger documentation sets. The system tracks which external files are used, when they were last verified, and whether they have changed since the last import.

**Build Systems Incorporating External References** - Continuous integration pipelines can query the MQSAQL index during documentation builds to fetch the latest versions of referenced external files or to validate that cached copies remain consistent with the originals.

**Research Projects with Extensive Bibliographies** - Academic and technical researchers can organize their collection of external technical documents, preprints, and supplementary materials using the index, enabling cross-referencing and citation management.

**Internal Knowledge Base Maintenance** - Organizations with distributed internal documentation across multiple repositories can use MQSAQL to create a unified view of referenced content, reducing duplication and ensuring consistency.

**Compliance and Audit Trail Creation** - Regulated environments can leverage the integrity verification features to demonstrate that externally referenced content has not been altered after being incorporated into audited documentation.

## 快速开始

```bash
git clone https://github.com/vhjhbkhggk/mqsaql.git
cd mqsaql
pip install -e .
mqsaql init --index-path ./index.json
mqsaql add --source https://github.com/vhjhbkhggk/mqsaql/blob/main/d05afbc71c62a2f0.md
mqsaql build --output ./docs/references.json
```

## 贡献指南

**提交问题报告** - 在提交问题之前，请先搜索现有的issue以避免重复。问题报告应包含完整的复现步骤、预期行为与实际行为的对比，以及相关的环境信息（操作系统、Python版本、MQSAQL版本）。所有问题报告均需使用提供的issue模板。

**创建拉取请求** - 拉取请求应当聚焦于单一功能或修复。提交前请确保所有测试通过（运行pytest tests/），代码已格式化（运行black src/ tests/），并且静态类型检查无错误（运行mypy src/）。拉取请求描述应详细说明变更内容、动机和测试覆盖情况。

**代码风格规范** - 项目遵循PEP 8编码规范，使用Black格式化工具保持一致的代码风格，使用isort管理导入语句顺序。所有公共API必须包含完整的docstring，遵循Google风格注释规范。

**测试覆盖要求** - 所有新增功能必须包含对应的单元测试，修复的bug应包含回归测试用例。测试覆盖率不得低于百分之八十，持续集成流水线会在每次提交时自动检查覆盖率。

**提交信息规范** - 提交信息采用约定式提交格式，使用feat、fix、docs、style、refactor、perf、test、chore等类型前缀。提交信息主体应使用祈使语气，详细说明变更的原因和影响。

## 常见问题

**问：MQSAQL如何确保外部参考文件的完整性？**

答：MQSAQL在首次添加外部参考文件时计算其内容的SHA-256校验和，并将该校验和与文件元数据一同存储。在后续的build或verify操作中，系统会重新计算远程文件的实际校验和并与存储值进行比对。如果校验和不匹配，系统会发出警告并记录差异，用户可以选择更新索引或手动审查变更。这种机制可以有效检测外部参考文件在未通知情况下被修改的情况。

**问：如何处理外部参考文件被删除或移动的情况？**

答：当执行build或update命令时，MQSAQL会尝试访问每个已索引的URL。如果收到HTTP 404或410状态码，系统会将相应条目标记为失效状态并记录在日志中。用户可以使用mqsaql list --stale命令查看所有失效条目，并使用mqsaql remove --stale批量移除它们，或者使用mqsaql update --url <old-url> --new-url <new-url>更新URL映射。系统不会自动删除失效条目，以防止意外丢失引用关系。

**问：MQSAQL索引文件能否与版本控制系统配合使用？**

答：索引文件采用JSON格式存储，完全适合提交到Git等版本控制系统中。推荐的做法是将索引文件作为项目的一部分提交，这样所有团队成员都能获得一致的外部参考视图。当团队成员添加或更新外部参考时，索引文件的变更会通过正常的版本控制流程进行审查和合并。但需要注意，索引文件中存储的校验和是基于特定时间点的外部内容计算得出的，不同时间点克隆项目的用户可能会因外部内容更新而收到校验和警告，这是预期行为，用于提醒用户内容已变更。
