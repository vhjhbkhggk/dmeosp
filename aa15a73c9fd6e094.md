# ResourceBridge

ResourceBridge 是一个面向开发者与技术研究人员的结构化外链与参考文献聚合工具。项目定位为轻量级技术资源导航枢纽，通过对散落于代码仓库、技术博客、官方文档中的高质量外部链接进行人工筛选与分类索引，帮助用户快速定位特定领域的关键资料，避免重复检索与信息过载。

ResourceBridge 本身不存储任何受版权保护的内容，仅提供指向公开互联网资源的稳定引用链接。项目适用于技术文档撰写者、开源项目维护人、技术课程设计者以及需要系统化整理阅读列表的研究人员。通过统一的 Markdown 索引格式与自动化校验脚本，ResourceBridge 确保所有外链在发布前均通过可达性检查，并定期对失效链接进行标记与替换。

## 功能概览

- **分类索引体系**：按技术领域、资源类型、适用水平对每个外链进行多维度标签标记，支持快速筛选。
- **批量链接校验**：内置基于 Python 的链接可达性检测脚本，可自动识别 404、超时、重定向等异常状态。
- **Markdown 原生渲染**：所有资源列表以标准 Markdown 格式呈现，兼容 GitHub、GitLab、Gitee 等主流代码托管平台的渲染引擎。
- **版本化更新记录**：每次增删改链接均通过 Pull Request 提交，保留完整的变更历史与审核轨迹。
- **自定义标签扩展**：用户可根据自身需求在本地派生项目中增加自定义标签字段，不影响上游更新合并。
- **链接去重与归一化**：自动检测重复提交的 URL，并基于域名与路径进行归一化比对，避免冗余条目。
- **快速全文检索**：借助 GitHub 原生搜索能力，可对资源标题、描述、标签进行全文检索。
- **批量导入导出**：支持从 CSV 与 JSON 格式批量导入链接清单，也可将当前索引导出为标准书签文件。

## 应用场景

1. 技术文档编写过程中的参考资料管理。当编写系统设计文档或技术方案时，可直接从 ResourceBridge 中引用经过验证的外部链接，减少手动搜索与验证时间，同时保证引用的权威性与时效性。

2. 开源项目 README 与 Wiki 的扩展资源维护。项目维护者可将 ResourceBridge 作为子模块引入，在项目的文档中嵌入特定分类的资源链接，避免在主仓库中维护大量外部引用造成的冗余。

3. 技术培训与课程设计中的阅读清单整理。讲师或课程设计者可以利用 ResourceBridge 的分类标签快速生成针对特定主题（如数据库原理、分布式系统、前端工程化）的推荐阅读列表，并导出为结构化文档。

4. 个人技术知识库的外链备份与共享。独立研究者或工程师可将个人收藏的优质技术文章、官方文档、工具站通过 ResourceBridge 进行统一归档，并通过 GitHub 公开分享给同行。

5. 团队内部技术雷达的素材来源。技术委员会或架构组可定期从 ResourceBridge 的更新记录中提取新兴技术相关资源，作为技术选型评估的参考依据。

## 快速开始

以下操作假设您已安装 Git 与 Python 3.8 及以上版本。

```bash
git clone https://github.com/your-org/resource-bridge.git
cd resource-bridge

pip install -r requirements.txt

python scripts/check_links.py --source ./resources --output ./reports/link_status.json
```

首次运行校验脚本后，系统会在 `./reports` 目录下生成链接状态报告。若需启动本地预览服务，可执行：

```bash
python -m http.server 8000 --directory ./docs
```

随后在浏览器中访问 `http://localhost:8000` 查看索引页面的静态渲染效果。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.8 及以上 | 用于运行链接校验脚本与索引生成工具 |
| Git | 2.25 及以上 | 用于克隆仓库及提交变更记录 |
| requests | 2.28.0 及以上 | 处理 HTTP 请求，用于链接可达性检测 |
| beautifulsoup4 | 4.11.0 及以上 | 解析 HTML 元数据，辅助提取页面标题与描述 |
| markdown | 3.4.0 及以上 | 将 Markdown 索引渲染为静态 HTML 预览页面 |
| pytest | 7.2.0 及以上 | 运行单元测试，保证校验逻辑的正确性 |
| pre-commit | 2.21.0 及以上 | 本地 Git 钩子管理，用于提交前自动执行格式检查与链接校验 |

## 贡献指南

1. 在 GitHub 上 Fork 本仓库，并在本地克隆派生副本。请确保派生仓库的同步分支策略与上游保持一致，避免提交冲突。
2. 在 `resources/` 目录下找到对应批次文件夹，或新建批次文件夹（按顺序编号）。新增或修改资源链接时，需同时更新 `index.json` 中的标签与摘要信息，保持索引与内容一致。
3. 所有新增链接必须通过本地链接校验脚本 `python scripts/check_links.py --source ./resources --output ./reports/link_status.json`，确保返回码为 0 且无严重错误。校验失败的单条链接将被标记为阻塞项，不允许合并。
4. 提交变更时请遵循语义化提交规范，使用 `feat:`、`fix:`、`docs:` 等前缀，并在提交信息中明确说明所涉及的批次编号与链接数量。提交前 pre-commit 钩子将自动运行格式检查与基本校验。
5. 向上游仓库发起 Pull Request，描述变更动机、涉及批次范围以及校验报告摘要。至少需要一名维护者进行审核，审核通过后方可合并。合并后 GitHub Action 将自动触发全量链接重新校验。

## 常见问题

Q: 如果发现某个链接已经失效，应该如何处理？

A: 首先通过人工确认该链接是否已迁移至新地址。若存在有效重定向，则更新为最终目标 URL；若完全失效且无法找到替代资源，则应在 `index.json` 中将该条目标记为 `deprecated` 并在备注中注明失效日期与原因。标记为废弃的链接不会从列表中物理删除，但会在渲染时添加删除线样式，以保持引用历史的完整性。

Q: 可以提交非技术类或商业化推广性质的链接吗？

A: ResourceBridge 明确不接受任何形式的商业化推广链接、付费课程入口、带有联盟跟踪参数的 URL 以及内容质量低于社区平均水平的个人博客。所有提交的链接应具备明确的技术参考价值，包括但不限于官方文档、知名开源项目、经过同行评审的技术论文、以及被广泛引用的技术问答。维护者保留对链接内容质量的最终裁量权。

Q: 如何将 ResourceBridge 的索引集成到我的项目文档中？

A: 推荐使用 Git Submodule 方式将本仓库作为子模块添加到您的项目中，然后在您的 README 或文档中通过相对路径引用特定批次或全量索引文件。您也可以定期运行导出脚本生成纯文本或 JSON 格式的摘要，将其复制到您的项目目录中。对于需要实时同步的场景，建议在您的 CI 流水线中添加对 ResourceBridge 仓库的拉取步骤。
