# ResourceIndex MQS

ResourceIndex MQS 是一个面向技术研究、安全分析和数据溯源场景的轻量化外链索引与文档聚合系统。该项目不对仓库内的 Markdown 文件做任何内容层面的修改或渲染，仅提供结构化的索引视图与本地预览能力，帮助用户在海量散落的技术笔记、漏洞报告、配置文件片段与日志摘要中快速定位目标条目。

项目定位为技术资源的外链汇总站，适用于需要高频查阅外部参考材料、维护个人知识库或搭建团队内部文档门户的开发者与研究员。通过固定格式的命名规范与目录树组织，用户可离线持有完整的数据集，并借助 MQS 提供的索引机制在数十万级条目中执行高效的路径检索与上下文关联。

## 功能概览

**静态文档索引生成**：扫描仓库内 main 分支下的全部 Markdown 文件，提取文件名中的十六进制特征值与基础元数据，构建只读索引表。

**本地轻量级 HTTP 服务**：基于 Python 标准库提供即开即用的预览服务器，支持通过命令行参数绑定主机地址与端口号。

**文件路径模糊匹配**：支持基于文件名前缀、中缀、后缀的快速过滤，返回匹配条目的完整 GitHub 原始链接。

**批次化资源分组**：将当前批次（92/114）的 180 个资源链接自动归入批次视图，便于增量更新与变更追踪。

**纯文本元数据缓存**：索引结果以纯文本形式写入本地缓存目录，避免重复扫描，提升二次启动速度。

**可插拔的输出格式化器**：内置 JSON、CSV、Markdown 表格三种输出格式，满足不同下游工具链的消费需求。

## 应用场景

安全分析人员在处理外部威胁情报时，需频繁引用 GitHub 上分散的临时性分析片段。MQS 可将这些片段的链接统一收录并按批次组织，分析人员仅需记住批次编号即可在本地快速检索对应条目，无需反复浏览 GitHub 文件列表。

技术文档编写者在整理项目归档时，往往需要同时引用多个外部仓库中的特定 commit 版本文件。通过 MQS 的路径模糊匹配功能，编写者可基于已知的十六进制前缀一键拉取同批次所有相关链接，批量插入文档的外链附录。

团队内部进行代码审计复盘时，审计报告常涉及大量指向外部临时存储库的参考链接。MQS 允许审计团队将链接按审计轮次分批导入，每批独立维护索引，确保不同轮次的引用数据互不干扰且可追溯。

## 快速开始

执行以下命令完成项目克隆、依赖安装与本地服务的启动。

```bash
git clone https://github.com/vhjhbkhggk/mqsaql.git
cd mqsaql
pip install --no-cache-dir -r requirements.txt
python -m mqs.server --host 127.0.0.1 --port 8080
```

服务启动后，访问 http://127.0.0.1:8080 即可查看当前批次的索引首页。索引数据默认从仓库根目录的 main 分支读取，无需额外配置。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 及以上 | 核心运行环境，提供 HTTP 服务与文件扫描能力 |
| pip | 21.0 及以上 | 依赖项安装工具，用于拉取 requirements.txt 中声明的第三方库 |
| Git | 2.25 及以上 | 用于克隆仓库以及后续拉取上游更新 |
| 文件系统 | ext4 / NTFS / APFS | 要求支持长文件名（255 字节以上），用于存储十六进制命名的 Markdown 文件 |
| 内存 | 512 MB 及以上 | 索引构建过程中缓存文件列表与元数据，大仓库场景建议 1 GB |
| 网络 | 任意 | 仅用于首次克隆与后续拉取更新，本地预览服务不依赖外网 |
| 操作系统 | Linux / macOS / Windows | 跨平台支持，路径分隔符自动适配 |
| 终端 | 支持 UTF-8 编码 | 用于正确显示索引输出的十六进制字符串与特殊字符 |
| 浏览器 | 现代浏览器（Chrome / Firefox / Edge） | 用于访问本地预览服务的管理界面 |

## 项目结构

项目采用扁平化顶层设计，核心模块集中于 mqs 包内，配置与文档独立存放于根目录。

```
mqsaql/
├── mqs/                                # 核心 Python 包
│   ├── __init__.py                     # 包版本号与公开 API 声明
│   ├── server.py                       # HTTP 预览服务主入口，绑定路由与静态资源
│   ├── indexer.py                      # 文件扫描与索引构建逻辑，包含批次过滤器
│   ├── matcher.py                      # 文件名模糊匹配与正则表达式过滤器
│   ├── formatter.py                    # JSON / CSV / Markdown 输出格式转换器
│   ├── cache.py                        # 纯文本元数据缓存读写与过期校验
│   └── cli.py                          # 命令行参数解析与子命令分发
├── docs/                               # 文档目录，包含完整的使用手册与参考
│   ├── quickstart.md                   # 快速入门指南，含安装与首次运行步骤
│   ├── indexing.md                     # 索引机制深度说明，含扫描策略与冲突处理
│   ├── cli.md                          # 命令行参数完整列表与使用示例
│   ├── formats.md                      # 三种输出格式的字段定义与解析示例
│   └── maintenance.md                  # 缓存清理、索引重建与版本迁移操作指南
├── tests/                              # 单元测试与集成测试用例
│   ├── test_indexer.py                 # 索引扫描功能的覆盖率测试
│   ├── test_matcher.py                 # 模糊匹配算法的边界条件测试
│   └── test_formatter.py               # 输出格式器的数据一致性测试
├── scripts/                            # 辅助脚本，用于批量操作与数据校验
│   ├── validate_links.py               # 校验资源列表中的链接是否可访问
│   └── batch_import.py                 # 从外部 CSV 批量导入链接并生成索引
├── config/                             # 配置文件目录
│   └── index.yaml                      # 索引参数配置，含批次号、缓存路径与输出格式
├── requirements.txt                    # Python 第三方依赖声明（仅含标准库扩展）
├── setup.py                            # 包安装脚本，支持 pip install -e 模式
├── LICENSE                             # MIT 许可证全文
└── README.md                           # 项目根文档（即本文档）
```

## 贡献指南

1. 复刻主仓库至个人账户，在本地新建功能分支。分支命名建议采用 `feature/` 或 `fix/` 前缀，后接简短描述。

2. 在 `mqs/` 目录下完成代码修改或新增功能后，请同步更新 `docs/` 下对应的文档文件，确保命令行参数、配置项与输出格式的描述与实际行为一致。

3. 提交前运行 `tests/` 目录下的全部测试用例，确保既有的索引、匹配与格式化功能未被破坏。新增功能须附带对应的测试方法。

4. 提交 Pull Request 至主仓库的 main 分支，请求描述中需注明本次变更的影响范围、测试结果以及文档更新情况。

5. 审核通过后，由维护者合并至 main 分支并更新版本号。贡献者将列入项目鸣谢列表。

## 常见问题

问：启动服务后浏览器无法访问本地页面，可能是什么原因？

答：请检查绑定的主机地址是否为 127.0.0.1 或 0.0.0.0。若使用 127.0.0.1，仅允许本机访问；若需局域网内其他设备访问，请改为 0.0.0.0 并确认防火墙已放行对应端口。同时确认终端未报端口占用错误，若 8080 端口被占用，可通过 --port 参数指定其他端口。

问：索引页面显示的条目数量与预期不符，部分文件未被收录如何处理？

答：首先确认文件位于仓库根目录的 main 分支下，且文件扩展名为 .md。索引器默认跳过隐藏文件（以点开头）以及非 Markdown 后缀的文件。若文件符合上述条件但仍未收录，可检查文件名是否包含非法字符或长度超出操作系统限制。使用 `python -m mqs.cli rebuild` 命令强制重建索引缓存可解决大部分缓存不一致问题。

问：如何导出当前批次的所有链接为 CSV 格式，用于导入其他数据分析工具？

答：通过命令行参数指定输出格式即可。执行 `python -m mqs.cli export --format csv --batch 92 --output links.csv`，索引器会将当前批次的全部链接以 CSV 格式写入指定文件。CSV 包含文件名、完整链接、文件大小与最后修改时间四个字段，可直接用 Excel 或 Pandas 加载。
