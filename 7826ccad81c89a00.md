# RepoIndex

RepoIndex 是一个面向开发者与技术研究者的结构化外链与资源聚合工具。该项目不直接存储文档或代码包，而是通过索引机制将分散于各处的技术资源（包括 GitHub 仓库中的 Markdown 笔记、技术规范、配置示例与操作手册）统一收录，并提供可检索、可分类、可版本追踪的访问入口。

RepoIndex 的目标用户包括架构师、运维工程师、技术调研人员以及需要长期维护外部依赖信息清单的项目管理者。该项目帮助用户快速定位特定主题的参考资料，避免重复检索，降低信息遗漏风险，同时通过统一的 URL 清单形式串联跨仓库、跨站点的零散资源。

## 功能概览

批量外链导入：支持以 Markdown 列表、CSV 或 JSON 格式导入外部链接，自动解析并归入对应分类。

资源分类标签：每条索引记录可标记多个自定义标签，支持按标签筛选与组合查询。

版本快照记录：每次更新索引时自动记录时间戳与变更摘要，便于回溯历史状态。

链接可用性检查：内置简单的 HTTP 头检测机制，可标记失效或重定向链接。

全文元数据检索：基于链接标题、来源仓库、文件路径与描述文本进行关键词匹配。

导出与集成：索引数据可导出为纯 Markdown 表格、JSON API 格式或用于静态站点生成的 YAML 文件。

## 应用场景

技术选型调研阶段，团队需要同时查阅多个开源仓库的文档、示例代码与版本发布说明。通过 RepoIndex 将相关资源链接集中管理，可减少反复切换浏览器标签与搜索关键词的时间消耗。

微服务架构下的配置中心或服务注册表文档分散在不同代码仓库的 /docs 目录中。使用 RepoIndex 建立统一的配置参考索引，可在发布前快速核对所有依赖项的文档版本与变更记录。

对外输出技术方案或审计报告时，需要附上所有参考资料的原始出处。RepoIndex 支持一键生成带分类和描述的引用清单，满足合规与溯源要求。

## 快速开始

以下命令演示如何获取 RepoIndex 源码、安装依赖并启动本地索引服务。

```bash
git clone https://github.com/your-org/repoindex.git
cd repoindex
pip install -r requirements.txt
python index_server.py --init --data ./data/sample_index.json
```

执行上述命令后，服务默认监听 127.0.0.1:8080。访问 http://127.0.0.1:8080 可查看索引首页，通过 /api/links 获取全部资源记录的 JSON 响应。

## 安装要求

| 依赖 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 或更高 | 核心运行时，用于解析索引、提供 API 服务 |
| pip | 21.0+ | Python 包管理工具，用于安装项目依赖 |
| Flask | 2.2.0+ | Web 服务框架，提供 REST API 与简易管理界面 |
| requests | 2.28.0+ | 用于执行链接可用性检查与元数据抓取 |
| PyYAML | 6.0+ | 用于支持 YAML 格式的索引导出与导入 |
| markdown | 3.4.0+ | 用于将资源描述字段渲染为 HTML 预览 |
| pytest | 7.2.0+ | 可选依赖，用于运行单元测试与集成测试 |

## 常见问题

Q: RepoIndex 是否会自动抓取每个链接的完整页面内容？
A: 不会。RepoIndex 只存储链接本身以及通过 HTTP 头获取的状态信息，不抓取或缓存页面正文，以保持轻量级并尊重目标站点的带宽资源。

Q: 索引数据可以存储在数据库而非 JSON 文件中吗？
A: 当前版本以 JSON 文件作为主要存储方式，方便版本控制与轻量部署。后续路线图计划支持 SQLite 和 PostgreSQL 作为可选存储后端。

Q: 如何更新已有索引中的标签或描述字段？
A: 通过 API 的 /api/update 接口传入资源 ID 与待修改字段，或在管理界面中编辑对应条目。每次修改会生成新的快照记录，旧版本仍可在 snapshots 目录中访问。
