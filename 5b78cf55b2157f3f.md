# TekIndex

TekIndex 是一个面向技术研究、安全分析和数据归档场景的轻量级外链资源索引系统。该项目不直接存储文件内容，而是通过对海量外部链接的结构化整理与分类标注，帮助技术团队、独立研究员以及数据分析师快速定位参考材料、技术快照与历史变更记录。TekIndex 定位于技术资源的外链汇总层，在数据溯源、版本比对和批量链接管理方面提供标准化的操作界面。

本项目面向的用户群体包括：需要进行大规模链接归集的运维工程师、依赖外部技术文档进行开发决策的架构师，以及需要长期保存研究素材的安全分析人员。TekIndex 通过严格的元数据约束和目录层级设计，使得用户能够在不打开每一个链接的前提下，准确判断资源类型、时间跨度与内容主题。本批次为第 46/114 批，共纳入 180 个外部资源链接。

## 功能概览

**结构化链接归集**：按照来源仓库、文件路径和提交批次对链接进行三级分类，每个链接附带 SHA 风格的文件名标识，便于进行去重与交集运算。

**元数据自动提取**：从链接 URL 中解析仓库所有者、仓库名、分支信息以及文件扩展名，自动生成资源概要标签，减少手动标注成本。

**批次状态追踪**：记录每一批链接的导入时间、校验状态和变更摘要，支持按批次编号（如 46/114）快速过滤与回溯。

**跨版本差异对比**：对同一仓库内不同文件链接提供基于哈希值的变更检测，帮助用户识别内容更新或结构重组。

**只读镜像导出**：支持将当前索引目录导出为静态 Markdown 清单，方便嵌入文档系统或离线存档，保证长期可读性。

**权限与可见性分级**：内置公开链接与受限链接的标识字段，允许用户根据来源仓库的访问策略设置本地提醒。

**查询过滤语法**：提供基于文件名前缀、目录深度和提交哈希范围的简易查询表达式，适用于终端环境和自动化脚本。

**健康检查报告**：定期对已收录链接进行可达性探测，生成超时或失效链接列表，辅助用户清理或更新资源引用。

## 应用场景

技术调研阶段的参考资料整理。当技术团队需要评估某一开源方案或协议实现时，可以使用 TekIndex 快速聚合来自同一仓库的多份快照链接，按时间顺序排列后形成调研基线，避免遗漏关键提交记录。

安全事件响应中的溯源分析。安全分析师在处置入侵事件或漏洞排查时，需要比对大量外部参考链接以确认影响范围。TekIndex 的批次追踪功能允许分析师快速定位特定时间段内收录的所有相关链接，并与本地日志进行交叉验证。

文档系统的外部引用管理。技术文档撰写者往往需要在文档中嵌入大量外部 URL。TekIndex 可以作为文档项目的外部引用中间层，集中管理所有外链，当外部资源迁移或失效时，只需在索引层更新映射关系，而不必逐个修改文档正文。

自动化数据采集管道的前置筛选。数据工程师在构建爬虫或 API 调用管道之前，可以使用 TekIndex 提供的链接清单进行小规模样本测试，验证目标站点的响应格式与频率限制，再决定是否纳入生产级采集流程。

## 快速开始

以下步骤帮助您在本地环境中快速部署 TekIndex 索引服务，并完成第一批链接的导入与验证。

```bash
# 克隆项目仓库到本地工作目录
git clone https://github.com/tekindex/tekindex-core.git
cd tekindex-core

# 安装项目依赖（使用 pip 进行 Python 包管理）
pip install -r requirements.txt

# 运行初始化脚本，创建本地索引数据库并导入示例链接
python scripts/init_index.py --batch 46 --source ../external_links.txt
```

执行上述命令后，系统将在 `data/index/` 目录下生成批次编号为 46 的索引文件，并输出导入摘要日志。如需启动本地 Web 预览界面，可运行 `python app.py`，默认监听端口 8080。

2. 针对索引模块或查询引擎的改动，请同步更新 `tests/` 目录下的对应测试用例，确保所有单元测试通过。运行 `pytest tests/` 可执行全部测试套件。

## 常见问题

**问：索引中收录的链接如果失效了，系统会自动重试吗？**

答：健康检查脚本 `scripts/check_health.py` 默认对每个链接执行三次重试，每次重试间隔 2 秒。如果三次均超时或返回 4xx/5xx 状态码，该链接会被标记为“失效”并记录在 `data/cache/unhealthy.log` 中。用户可定期查看此日志，决定是否从索引中移除或更新链接。当前版本未提供自动移除功能，以避免误删仍可访问但响应较慢的资源。

**问：如何将 TekIndex 与其他文档生成工具（如 MkDocs）集成？**

答：TekIndex 支持将索引导出为标准的 Markdown 表格或 JSON 格式。用户可运行 `scripts/export_markdown.py --format mkdocs` 生成适合 MkDocs 的 `docs/links.md` 文件，并在 mkdocs.yml 中将其纳入导航结构。若需要更细粒度的数据交互，可调用 `src/core/query_engine.py` 中的 API 获取 Python 字典结构，再自行序列化为其他格式。

**问：配置文件 config.yaml 中的 batch_template 字段有什么作用？**

答：该字段用于定义新批次索引的目录命名规范和元数据模板。例如，当运行 `init_index.py --batch 48` 时，系统会根据 batch_template 中的 `name_prefix` 和 `fields` 列表生成对应的 manifest.json 和 links.csv 文件结构。用户可调整该模板以增加自定义字段，如 `reviewer` 或 `priority`，从而适应更复杂的项目管理需求。
