# ResourceBridge

ResourceBridge 是一个面向开发者与技术研究人员的结构化外链资源归集与导航工具。它不生产内容，而是通过严谨的目录分类、版本追踪与元数据标注，将散落在互联网各处的优质技术文档、教程、规范、工具链页面以及社区讨论聚合为可检索、可追溯的本地知识索引。本项目适用于需要系统化整理外部学习资料、维护项目依赖文档或构建内部技术文库的团队与个人。

本项目解决的核心问题是：当技术团队或个人维护者需要引用大量外部资源时，如何保证链接的可溯源性、分类的合理性以及更新记录的可审查性。ResourceBridge 提供一套完整的 Markdown 索引模板、自动化校验脚本以及可视化导航页面生成方案，帮助用户从杂乱的浏览器书签或零散的笔记中解脱出来，形成长期可维护的资源资产。

## 功能概览

**结构化目录模板** 内置三级分类体系（基础规范、语言生态、工程实践等），用户可直接套用或自定义扩展，快速建立资源树形结构。

**链接存活校验** 集成基于 Python 的异步 HTTP 检查模块，可定期扫描索引文件中的外链，标记失效或变更的 URL 并生成报告。

**元数据增强标注** 支持为每条资源记录添加来源站点、更新日期、内容摘要、阅读优先级与标签，便于后续过滤与排序。

**多格式导出** 支持将索引数据导出为静态 HTML 导航站、JSON API 或纯文本清单，适配不同发布与集成场景。

**变更历史追踪** 结合 Git 版本控制，每一次资源的新增、删除或分类调整均可追溯到提交记录与变更说明，便于团队协作审核。

**模糊检索与标签过滤** 内置轻量级前端搜索组件，支持按标题关键词、标签集合以及内容摘要进行全文检索，并可按优先级或更新时间排序。

## 应用场景

**技术团队内部知识库构建** 团队可将日常开发中参考的官方文档、API 手册、最佳实践文章统一归集至 ResourceBridge 索引，并在每周例会中评审更新，确保所有成员访问的是最新且经过验证的资源版本。

**开源项目外链维护** 开源项目维护者常需要在 README 或 Wiki 中引用大量外部链接。使用 ResourceBridge 可独立维护一份外链清单，通过自动校验脚本定期检查链接可用性，避免项目文档中出现死链。

**个人学习路径管理** 自学者可将不同技术方向（如前端框架、后端中间件、DevOps 工具）的学习资料分类存入 ResourceBridge，并标注优先级与阅读状态，形成清晰的学习路线图，方便阶段性回顾。

**技术文档站的外挂索引** 若团队已拥有静态文档站（如 VuePress、Docusaurus），可将 ResourceBridge 生成的 JSON 数据通过接口挂载至文档站侧边栏或资源页面，实现外部资源的统一展示与跳转。

## 快速开始

以下步骤将在本地环境拉取项目代码、安装依赖并启动开发服务。

```bash
# 克隆仓库
git clone https://github.com/your-org/resource-bridge.git
cd resource-bridge

# 安装依赖（使用 Python 虚拟环境）
python -m venv venv
source venv/bin/activate  # Windows 下使用 venv\Scripts\activate
pip install -r requirements.txt

# 初始化示例索引并启动本地服务
python scripts/init_index.py --sample
python app.py --host 127.0.0.1 --port 8080
```

启动成功后，访问 `http://127.0.0.1:8080` 即可查看示例导航页面。若要自定义资源数据，请编辑 `data/index.yaml` 文件，然后运行 `python scripts/build.py` 重新生成静态页面。

| pytest | 7.0 及以上 | 可选依赖，用于运行单元测试与校验脚本 |

## 常见问题

**Q: 索引文件中的资源链接过多时，校验脚本是否会超时或内存溢出？**

A: 校验模块默认使用异步 IO 并发控制，最大并发数可在 `config.yaml` 中调整（默认 50）。对于超过 1000 条链接的索引，建议分批次校验并通过 `--batch-size` 参数控制每批处理数量。内存占用通常不超过 512MB。

**Q: 能否与现有的静态站点生成器（如 Hugo 或 VuePress）集成？**

A: 可以。ResourceBridge 提供 `build.py --export json` 命令，将完整索引数据输出为 JSON 文件。您可在静态站点生成器的数据加载阶段读取该 JSON，并将其渲染为侧边栏或独立资源页面。具体集成示例可参考文档中的 `docs/development/integration.md`。

**Q: 如何自定义链接校验的超时时间与重试策略？**

A: 在项目根目录创建 `config.local.yaml` 文件，覆盖 `checker` 字段下的 `timeout`、`retry_times` 与 `retry_delay` 参数。若未创建该文件，系统将使用 `config.default.yaml` 中的默认值（超时 10 秒，重试 2 次，延迟 1 秒）。注意 `config.local.yaml` 已被 `.gitignore` 忽略，适合存放本地个性化配置。
